安装cobbler问题

开始8上面没有cobbler，使了源码安装的方式，引发了不少报错，参考第一节，后来直接dnf安装，就不存在这些报错了。

客户端虚拟机BIOS启动设置

装的时候，由于esi虚拟机默认EFI启动，启动了n次都发现进入界面后，它自己又回到蓝色的那个网络启动界面，也看不出来什么报错，后来改成BIOS启动（建虚拟机的时候就可以改），才正常进入界面，看到报错信息

安装界面

上面配置好后，创建空白虚拟机，会看到这个界面，选下面这一个，就可以自动安装系统了

有两个选项，一个是local，默认是原来的系统。另一个是Centos，就是要装的。但是实际场景机器很多，不可能每一台都接显示器选一下。可以修改

/var/lib/tftpboot/pxelinux.cfg/default 文件，将local选项去掉,只保留我们要装的那一个，客户端开机后就进入这个指定的系统，而不会进入原装的系统了。安装完成后，尽量保留这个参数或者关机，防止其他机器关机重启后自动进入被重装系统！

安装完成后，在客户端会有一个anaconda-ks.cfg文件，这个文件和上面ks=url…的配置是相同的。

cobbler命令

  • cobbler commands 介绍
  • cobbler check 核对当前设置是否有问题
  • cobbler list 列出所有的cobbler元素
  • cobbler report 列出元素的详细信息
  • cobbler sync 同步配置到数据目录,更改配置最好都要执行下
  • cobbler reposync 同步yum仓库
  • cobbler distro 查看导入的发行版系统信息
  • cobbler system 查看添加的系统信息
  • cobbler profile 查看配置信息

设置固定ip地址

在批量部署时，为了方便，需要固定被装机器的ip地址，防止ip地址发生改变。

  • –name 随便起个名字
  • --mac 需要提前知道mac地址
  • –static 静态ip地址
  • –hostname 主机名
  • –ip-address ip地址

ks文件

ks文件可以做很多配置，很重要

  • 关闭防火墙
  • 关闭selinux权限
  • 进行分区
  • 大小是M
  • 要注释掉原来的autopart
